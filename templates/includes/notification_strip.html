{% load i18n %} {% comment %} Site-wide notifications strip (VERB-109). Rendered by base.html's {% block notifications %} on every page, below the nav and above the main content block. active_notifications comes from core.context_processors.notifications — already filtered to the current request's audience, enabled kill switch, and display window, and ordered highest priority first then newest. Each banner carries data-priority (neutral / low / normal / high) from the notification's priority_tone; src/css/main.css maps each tone to its colour (grey / blue / amber / red) so the colour mapping lives in one place. Each notification's content_sanitised was sanitised once at save() via core.services.sanitise_notification_html (nh3 allow-list) — this is the one audited exception to Invariant 4 (no mark_safe on user content), so rendering it with |safe here is deliberate and safe. Dismissal is client-side only (sessionStorage, no server round-trip): a dismissible notification's ID is recorded in sessionStorage on dismiss, and any element whose ID is already stored is hidden on load. A new browser session clears sessionStorage, so the notification reappears. Permanent (non-dismissible) notifications render no dismiss control and are never hidden by this script. {% endcomment %} {% if active_notifications %}
